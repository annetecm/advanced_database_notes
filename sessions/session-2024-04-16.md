## What I understood
- Full table scan: check all of the db registers, performance is affected
- SQL uses b-tree for indexes to avoid full table scans
- Index is an object stored on the db, it takes space on disk
- We do full index scan instead of full table scan because it is better in performance
- Hash takes less time but more space 
- Composite index: index composed by different columns
